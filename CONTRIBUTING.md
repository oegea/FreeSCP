# Contributing to FreeSCP

Thanks for your interest in FreeSCP.

## Issues — yes, please

Bug reports and feature ideas are very welcome. When filing a bug, include:

- macOS/Linux version and chip (Apple Silicon / Intel),
- the protocol and what you did,
- the relevant lines from the diagnostics log at `/tmp/winscp-native.log` (and, if it's a
  protocol/transfer issue, run with `WINSCP_SESSIONLOG=1` and attach `/tmp/winscp-session.log`).

## Code contributions — not accepted (for now)

FreeSCP is largely a personal, "vibe-coded" project. To keep it coherent and easy to reason about,
**pull requests are not being accepted at this time.** Please don't be offended if a code PR is
closed — it's nothing about you or the change.

You're of course free to **fork** it (it's GPLv3) and take it wherever you like.

## Privacy

There is no privacy policy and no data collection — FreeSCP is a local desktop app and a personal
project. It talks only to the servers you connect to.
